
### ✅ **1. RDS: Relational | DynamoDB: Non-Relational (Vendor Lock-in)**

- **RDS (Relational Database Service)**:
    
    - Fully managed relational database service by AWS.
    - Supports MySQL, PostgreSQL, SQL Server, Oracle, etc.
    - Best for applications that need structured data, ACID compliance, and SQL querying.

- **DynamoDB**:
    
    - AWS’s fully managed **NoSQL (non-relational)** database.
    - Highly scalable and fast.
    - Often used for serverless or real-time applications.
    - **Vendor lock-in** means it's tightly coupled with AWS; you can't easily migrate it to other cloud providers (like Azure or GCP).

- **Examples in other clouds**:
    - Azure: **Cosmos DB**
    - Google Cloud: **Firestore**

---

### ✅ **2. AWS Lambda = Azure Functions (FaaS / Serverless Services)**

- **AWS Lambda**:
    - A **Function-as-a-Service (FaaS)** offering.
    - You write code (function) and AWS runs it without you managing servers.
    - Only runs **on demand** (event-driven).
    - You pay **only when** the function runs.

- **Limitations**:
    - **Stateless**: Cannot store session data or use a local disk (no persistent disk storage).
    - Not ideal for workloads requiring **state**, such as video processing with intermediate files, unless paired with external storage (like S3)

---

### ✅ **3. Observability = Logging + Monitoring + Alerts**

- Observability helps you understand what's happening inside your system (app, infra) through:

    - **Logs**: Events and messages generated by apps (e.g., errors, HTTP requests).
    - **Monitoring**: Collecting metrics like CPU, RAM, disk I/O.
    - **Alerts**: Rules and notifications based on metrics or logs (e.g., alert when CPU > 80%).

---

### ✅ **4. “Datadog is Baap of Observability”**

- **Datadog**: A powerful SaaS tool used to monitor cloud infrastructure and applications.
    - Integrates with AWS, Azure, GCP, Kubernetes, Docker, etc.
    - Provides advanced features: dashboarding, anomaly detection, alerts, APM, and log analysis.
    - “Baap” (father) = slang meaning "best or most powerful" in this context.
---

### ✅ **5. Regional vs Global | CDN: Content Delivery Network**

- **CDN**:
    - A network of globally distributed **edge servers** that cache and deliver **static content** (HTML, CSS, JS, images, videos).
    - Speeds up delivery to users globally.
        
- **Regional vs Global**:
    - **VMs (Virtual Machines)**: Usually **regional**, meaning tied to a specific data center or region.
    - **CDNs / S3 / DNS (Route 53)**: **Global services**, designed to deliver content or route traffic worldwide efficiently.

---

### ✅ **6. Global Database**

- Refers to databases that:
    - Can **replicate** data across multiple regions.
    - Provide **low latency** and **high availability** globally.
- Examples:
    - **Aurora Global Database** (AWS)
    - **Cosmos DB** (Azure)
    - **Spanner** (Google Cloud)
