## 🚀 1. Core Idea (1-line)

👉 **Kubernetes control plane manages the entire cluster and ensures desired state is maintained**

---

## 🧠 2. Full Architecture Overview

Cluster =

### 🧠 Control Plane (Master)

- Decision maker


### 💪 Worker Nodes

- Run actual workloads


---

# ⚙️ 3. Control Plane Components (VERY IMPORTANT 🔥)

---

## 🟢 1. kube-apiserver (ENTRY POINT 🚪)

👉 MOST IMPORTANT COMPONENT

- All communication goes through this

- CLI (`kubectl`) talks to this

- Validates requests


💡 Example:

```bash
kubectl get pods
```

👉 Goes to API Server first

---

## 🟣 2. etcd (DATABASE 💾)

👉 Stores EVERYTHING

- Cluster state

- Configurations

- Secrets (⚠️ sensitive)


👉 Key-value store

💡 If etcd dies → cluster dies 💀

---

## 🔵 3. kube-scheduler (PLACEMENT ENGINE 🧠)

👉 Decides:

- Which pod goes to which node


Based on:

- Resources

- Affinity

- Taints


---

## 🟡 4. kube-controller-manager (AUTO HEALER 🔁)

👉 Runs controllers:

- Node controller

- ReplicaSet controller

- Job controller


👉 Ensures:

- Desired state = actual state


---

# 💪 4. Worker Node Components

---

## 🟢 1. kubelet

👉 Agent on each node

- Talks to API server

- Runs pods

- Ensures containers running


---

## 🔵 2. kube-proxy

👉 Handles networking

- Service routing

- Load balancing


---

## 🐳 3. Container Runtime

👉 Runs containers

Examples:

- containerd

- CRI-O


---

# 🔗 5. Full Flow (SUPER IMPORTANT 🔥)

```text
kubectl → API Server → etcd

API Server → Scheduler → Node selected

Controller Manager → ensures desired state

kubelet → runs pod on node

kube-proxy → handles traffic
```

---

## 💥 6. Real-world Flow Example

You run:

```bash
kubectl apply -f deployment.yaml
```

👉 What happens:

1. API server receives request

2. Stores in etcd

3. Scheduler assigns node

4. kubelet creates pod

5. kube-proxy exposes it


---

## ⚠️ 7. Common Mistakes

❌ Thinking scheduler runs pods  
👉 It only decides placement

❌ Ignoring etcd importance  
👉 It’s the backbone

---

## 💼 8. Interview Answer

👉 _“The Kubernetes control plane consists of components like API server, etcd, scheduler, and controller manager that manage cluster state, while worker nodes run workloads via kubelet, kube-proxy, and container runtime.”_

---

## 🧠 9. Memory Trick

👉 **API Server = gate 🚪**  
👉 **etcd = brain memory 💾**  
👉 **Scheduler = decision 🧠**  
👉 **Controller = correction 🔁**

---

## 🔥 10. Pro Insight (Real-world)

- Control plane must be:

- Highly available

- Backed up (etcd backups 🔥)
