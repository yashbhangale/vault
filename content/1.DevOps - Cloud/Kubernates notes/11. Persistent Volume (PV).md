## 🚀 1. Core Idea (1-line)

👉 **DaemonSet ensures that a Pod runs on EVERY node in the cluster**

---

## 🧠 2. Why DaemonSet Exists (VERY IMPORTANT ⚠️)

Normal controllers (Deployment, ReplicaSet):

- Run specific number of pods (e.g., 3, 5)


But sometimes you need:  
👉 1 pod per node

---

## 💡 3. Use Cases (REAL-WORLD 🔥)

DaemonSet is used for:

- 📊 Monitoring agents (Prometheus node exporter)

- 🧾 Logging agents (Fluentd, Logstash)

- 🔐 Security agents

- 🌐 Networking (CNI plugins)


👉 Basically: **node-level services**

---

## ⚙️ 4. How It Works

- New node added → pod automatically deployed

- Node removed → pod removed


👉 Fully automatic

---

## 🔥 5. Example Scenario

Cluster:

- 3 nodes


DaemonSet:  
👉 Will run **3 pods (1 per node)**

Add 1 node:  
👉 Now 4 pods automatically

---

## 📦 6. Example YAML

```yaml
apiVersion: apps/v1
kind: DaemonSet
metadata:
  name: node-monitor
spec:
  selector:
matchLabels:
  app: monitor
  template:
metadata:
  labels:
app: monitor
spec:
  containers:
  - name: monitor
image: node-exporter
```

---

## 🧠 7. Key Difference (IMPORTANT 🔥)

|Feature|Deployment|DaemonSet|
|---|---|---|
|Pod count|Fixed number|One per node|
|Scaling|Manual|Auto with nodes|
|Use case|Apps|Node services|

---

## 💥 8. Real-world DevOps Insight

In your stack (SigNoz, Prometheus 👀):

👉 Node exporter runs via DaemonSet  
👉 Collects metrics from each node

---

## ⚠️ 9. Advanced Concept (IMPORTANT)

👉 You can control where pods run using:

- Node selectors

- Taints & tolerations


Example:

- Run only on GPU nodes 🔥


---

## ⚡ 10. CKA Commands

```bash
kubectl get daemonsets
kubectl describe daemonset <name>
```

---

## 💼 11. Interview Answer

👉 _“DaemonSet ensures that a copy of a Pod runs on all or selected nodes in a Kubernetes cluster, commonly used for logging, monitoring, and node-level services.”_

---

## 🧠 12. Memory Trick

👉 **Deployment = count-based 📊**  
👉 **DaemonSet = node-based 🖥️**

---

## 🔥 13. Common Mistakes

❌ Using Deployment for node agents  
❌ Forgetting DaemonSet auto-scales with nodes  
❌ Not using tolerations for special nodes
