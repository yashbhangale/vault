# ☸️ Imperative Commands (kubectl Fast Ops)

> - Creating deployments & services quickly  
> - Using `--dry-run` to generate YAML

>
> 📌 VERY IMPORTANT for CKA (time-saving tricks)

## ⚡ Concept 1: Why Imperative Commands?

> [!important]

---

### 🔧 Use Cases:

- Quickly create test pods  
- Check connectivity  
- Validate network policies :contentReference[oaicite:0]{index=0}

## ⚙️ Imperative vs Declarative (Quick Recall)

| Type         | Meaning            |
|--------------|-------------------|
| Imperative   | HOW to do         |
| Declarative  | WHAT you want     |

---

> [!quote]
> Imperative = speed ⚡  
> Declarative = production 📜

## 🧪 Concept 2: Create Deployment Imperatively

> [!example]

```bash
kubectl create deployment backend-deploy \
  --image=hashicorp/http-echo \
  --replicas=3 \
  --port=5678 \
  --dry-run=client -o yaml > backend-deploy.yaml
```

---

### 🧠 Key Flags:

- `--image` → container image  
- `--replicas` → number of pods  
- `--port` → container port (doc purpose)  
- `--dry-run=client` → validate only  
- `-o yaml` → output YAML  

---

### 🎯 Benefit:
- Generates ready YAML → edit → apply  

---

## ✏️ Editing YAML (Important)

> [!warning]

- Remove unnecessary fields  
- Fix labels:
```yaml
labels:
  app: backend
```

---

- Add arguments if needed:

```yaml
args:
  - "-text=Hello from backend"
```

---

## 🚀 Apply Deployment

```bash
kubectl apply -f backend-deploy.yaml
```

---

## 🌐 Concept 3: Expose Deployment (ClusterIP)

> [!example]

```bash
kubectl expose deployment backend-deploy \
  --type=ClusterIP \
  --port=9090 \
  --target-port=5678 \
  --name=backend-svc \
  --dry-run=client -o yaml >> backend-deploy.yaml
```

---

### 🧠 Key Idea:

> [!quote]
> expose = create service from deployment

---

## 📦 Concept 4: Frontend Deployment

```bash
kubectl create deployment frontend-deploy \
  --image=nginx \
  --replicas=3 \
  --port=80 \
  --dry-run=client -o yaml > frontend-deploy.yaml
```

---

## 🌐 Concept 5: NodePort Service (Frontend)

```bash
kubectl expose deployment frontend-deploy \
  --type=NodePort \
  --port=80 \
  --target-port=80 \
  --name=frontend-svc \
  --dry-run=client -o yaml >> frontend-deploy.yaml
```

---

### ⚠️ Important:

> [!warning]
> NodePort is NOT auto-set in YAML

👉 Add manually:

```yaml
nodePort: 31000
```

---

## 🧪 Concept 6: Test Pod (VERY IMPORTANT)

> [!success]
> Used for debugging & testing connectivity

---

### ⚙️ Command:

```bash
kubectl run test-pod \
  --image=nginx \
  -it --rm \
  --restart=Never \
  --labels="app=test" \
  -- /bin/sh
```

---

### 🧠 Flags:

- `-it` → interactive terminal  
- `--rm` → delete after exit  
- `--restart=Never` → standalone pod  
- `--labels` → assign label  

---

## 🔗 Test Connectivity

```

```

## 🧠 Concept 7: DNS Resolution (CoreDNS)

> [!important]

---

### 📌 Inside Pod:

```bash
cat /etc/resolv.conf
```

---

### 🧠 Output:
- Nameserver → Cluster DNS IP  
- Usually:
```
10.96.0.10
```

---

### 🔍 Verify DNS Service:

```bash
kubectl get svc -n kube-system
```

---

### 🧠 Key Insight:

> [!quote]
> CoreDNS resolves service names → IPs

---

## 🔁 Flow Recap

> [!example]

```
Pod → backend-svc → CoreDNS → ClusterIP → Pod
```
