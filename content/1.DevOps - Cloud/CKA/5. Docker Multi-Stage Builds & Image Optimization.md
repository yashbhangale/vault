# 🐳 Day 5: Docker Multi-Stage Builds & Image Optimization

> [!summary] Video Overview
> This note explains the concept of **Docker Multi-Stage Builds**, how they help optimize image sizes, and why separating build and runtime environments is critical for production-ready containers.
>
> 📌 Core focus:
> - Build vs Runtime separation  
> - Image size optimization  
> - Practical Dockerfile implementation  

---

## 🍕 Concept 1: Build vs Runtime (Pizza Analogy)

> [!quote]
> Anything required to *build* the application should NOT be part of the final image.

Think of making a pizza:
- Build stage → Ingredients, tools (microwave, measuring cup)
- Runtime stage → Eating the pizza 🍕

### 🧠 Key Idea
- Build tools ≠ Runtime requirements
- Final image should only contain **what is needed to run the app**

---

## 📉 Concept 2: Why Smaller Images Matter

> [!important]
> Smaller Docker images are faster, safer, and more efficient.

### 🚀 Benefits:
- ⚡ Faster pull & push
- 🚀 Faster deployment
- 🔒 Reduced attack surface (more secure)
- 💾 Less storage usage

## ⚙️ Concept 3: Compiled vs Interpreted Languages

> [!info] Why this matters?
> Determines whether your app needs a **build stage**

### 🧱 Compiled Languages
- Require conversion to machine code
- Need build tools (compilers)
- Examples:
  - Java
  - C / C++
  - Go

### 🧾 Interpreted Languages
- Run line-by-line via interpreter
- No compilation required
- Examples:
  - Python
  - JavaScript
  - PHP

### 🧠 Takeaway
- Compiled → Heavy build stage required  
- Interpreted → Usually no build stage (but can still use multi-stage)

## 🐍 Concept 4: Example Application (Python Flask)

> [!example] Application Details
> - Framework: Flask  
> - Port: 5000  
> - Output: `Hello Docker`

---

## 📄 Single Stage Dockerfile

```dockerfile
FROM python:3.9

WORKDIR /app

COPY app.py /app

RUN pip install flask

EXPOSE 5000

ENTRYPOINT ["python"]
CMD ["app.py"]
```

---

## 🔧 Improvements in Dockerfile

> [!tip]
> Follow best practices for cleaner Dockerfiles

- Use `COPY` instead of `ADD`
- Use `ENTRYPOINT + CMD` together

---

## ⚠️ Problem: Unnecessary Build Tools

```bash
apt update
apt install -y build-essential
```

> [!warning]
> Build tools increase image size unnecessarily

### 📦 Result
- Image size: **~495 MB**

---

## 🏗️ Concept 5: Multi-Stage Dockerfile

> [!example] Multi-Stage Build

```dockerfile
# Stage 1: Build Stage
FROM python:3.9 AS build

WORKDIR /app
COPY app.py .

RUN apt update && apt install -y build-essential
RUN pip install flask

# Stage 2: Final Runtime Stage
FROM python:3.9

WORKDIR /app

COPY --from=build /app .

RUN pip install flask

EXPOSE 5000

ENTRYPOINT ["python"]
CMD ["app.py"]
```

---

## 🔑 Key Instruction

```dockerfile
COPY --from=build /app .
```

> [!important]
> Copies only required files from build stage → excludes heavy tools

- Same functionality ✅
