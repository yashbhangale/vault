## ⚙️ Concept 1: Imperative vs Declarative

> [!important]
> Two ways to configure any system (including Kubernetes)

---

### ⚡ Imperative Approach

> [!info]
> Tell the system **HOW to do something**

```bash
kubectl run mypod --image=nginx
```

- Step-by-step instructions
- Quick tasks
- Not scalable

---

### 📜 Declarative Approach

> [!info]
> Tell the system **WHAT you want**

```bash
kubectl apply -f pod.yaml
```

- Define desired state
- Kubernetes ensures it

---

### 🧠 Key Difference

| Approach     | Focus        | Use Case           |
|--------------|-------------|-------------------|
| Imperative   | How         | Quick tasks       |
| Declarative  | What        | Production setups |

---

## 🚀 Why Declarative is Preferred

> [!success]

### 1️⃣ Idempotency
- Apply same file multiple times → same result

---

### 2️⃣ Version Control
- Store configs in Git
- Track changes
- Team collaboration

---

### 3️⃣ Simplicity
- Human-readable configs
- Easy to understand

---

## 📄 Concept 2: What is YAML?

> [!important]
> YAML = Human-readable data format used in Kubernetes

---

### 🧠 Key Features:
- Easy to read & write
- Uses indentation (spaces ❗)
- Less verbose than JSON/XML

---

### ⚠️ Rules:
- ❌ No tabs (only spaces)
- Maintain consistent indentation
- Key-value structure

---

## 🧩 Concept 3: YAML Data Types

---

### 🔹 1. Scalar (Basic Values)

> [!example]

```yaml
name: "Prakash"
age: 21
height: 5.9
isEngineer: true
paymentDue: null
```

- String, Integer, Float, Boolean, Null

---

### 🔹 2. Dictionary (Map)

> [!example]

```yaml
address:
  flat: "X001"
  city: "Mumbai"
  country: "India"
```

- Collection of key-value pairs

---

### 🔹 3. List (Array)

> [!example]

```yaml
hobbies:
  - singing
  - dancing
  - traveling
```

---

## 🔥 Advanced YAML Concepts

---

### 🔁 Nested Dictionary

```yaml
contact:
  phone: "123456"
  email: "test@example.com"
```

---

### 📚 Nested List

```yaml
sports:
  - cricket
  - football
```

---

### 🧠 List with Dictionary

```yaml
education:
  degrees:
    - degree: "Bachelors"

    - degree: "Masters"

### 💼 List + Nested Dictionary

    role: "Consultant"
```

## 📦 Concept 4: Kubernetes Pod Manifest

> [!important]
> Every K8s object has 4 main fields:

### 🧠 KAMS Trick
- **A** → apiVersion  
- **K** → kind  
- **M** → metadata  
- **S** → spec  

---

## 🧾 Example Pod YAML

> [!example]

```yaml
apiVersion: v1
kind: Pod

metadata:
  name: mypod-2
  labels:
    app: nginx
    env: dev

spec:
  containers:
    - name: nginx-container
      image: nginx
      ports:
        - containerPort: 80

    - name: sidecar-container
      image: busybox
      args: ["sleep", "3600"]
```

---

## 🧠 Understanding Structure

> [!important]

- `spec` → Dictionary  
- `containers` → List  
- Each container → Dictionary  
- `ports` → List  
- `args` → List

## ⚙️ Concept 5: kubectl Commands

---

### 📌 Create Pod (Imperative)

```bash
kubectl run mypod --image=nginx
```

---

### 📌 Apply YAML (Declarative)

```bash
kubectl apply -f pod.yaml
```

---

### 📌 Get Pods

```bash
kubectl get pods
kubectl get pods -o wide
```

---

### 📌 Describe Pod

```bash
kubectl describe pod mypod-2
```

---

### 📌 Logs

```bash
kubectl logs mypod-2
kubectl logs mypod-2 -c sidecar-container
```

---

### 📌 Execute Command

```bash
kubectl exec mypod-2 -- ls
kubectl exec -it mypod-2 -- bash
```

---

### 📌 Delete Pod

```bash
kubectl delete pod mypod-2
```
